# Streamlit Cloud looks for a file with exactly this name, either next to
# the main app file or here at the repo root. It does NOT recognize
# requirements-all.txt by that alternate name, which is why the deployed
# app was failing with ModuleNotFoundError despite requirements-all.txt
# existing and working fine for local/manual installs.
#

# can bootstrap-train Module 1 and Module 2 on first deploy if their
# artifacts are missing (see ensure_pipeline_built() in module3/src/
# dashboard_common.py) — so the dashboard's runtime environment needs
# every module's dependencies available, not just module3's own.
#
# Versions pinned to match requirements-all.txt exactly. If you update one,
# update the other.
